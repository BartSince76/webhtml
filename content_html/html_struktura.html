Budowa i struktura dokumentu HTML

Dokument HTML

Dokument html to zwykły plik tekstowy składający się z [nazwy pliku].[rozszerzenia] przykład index.htm, index.html, który poza treśćią zawiera dodatkowe elementy zapisane w kodzie HTML.

HiperText Markup Language hipertekstowy język znaczników przeznaczony jest do tworzenia stron internetowych. Umożliwia zdefiniować wygląd i strukturę dokumantu html w tym nagłówki, paragrafy, obrazy, linki itp., dzięki czemu
przeglądarki internetowe potrafią renderować treść w sposób przystępny i zrozumiały dla odbiorcy końcowego.

Hipertekst to zbiór tekstów połączonych ze sobą odniesieniami, umożliwia swobodę w tworzenia treści i przekazu. Charakteryzuje się: niesekwencyjnością, nieliniowością, elastycznością.

Cechą charakterystyczną hipertekstu jest niesekwencyjność, która pozwala odbiorcy na nieliniową lekturę i stworzenie oryginalnego odczytania dzieła. Dzięki zastosowaniu hiperłączy w dokumentach html jako odnośników możliwe było opracowanie globalnego systemu tzw. sieci www (WorldWideWeb) umożliwiającej tworzenie rozbudowanych treści odwołujących się do rozmaitych źródeł rozprzestrzenionych na całym świecie.

Struktura dokumentu HTML

Szkielet struktury dokumentu HTML tworzą:

  1. <!DOCTYPE> | Deklaracja Typu Dokumentu <!DOCTYPE html>,
  2. HTML | Element główny tzw. korzeń <html>,
  3. HEAD | Nagłówka dokumentu (treść nagłówkowa),
  4. BODY | Ciało dokumentu (treść właściwa).

DTD | Deklaracja typu dokumentu <!DOCTYPE>

Prolog czyli Deklaracja Typu Dokumentu umieszczona zazwyczaj na początku dokumentu html składa się ze specjalnego bloku: <!DOCTYPE>. Określa w jakim standardzie został przygotowany dokument html dzięki czemu przeglądarka wie w jakim (?trybie) powinna zostać przetwarzać/renderować/interpretować strona.

Jeżeli dokument jest zgodny z wersją 4.01 możemy wykorzystać następujące deklaracje typu dokumentu:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd">

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd">

Obecnie najnowszy standard HTML5 zaleca stosowanie uproszczonej wersji deklaracji DTD

<!DOCTYPE html>

Tryb dziwactw
W trybie dziwactw przeglądarki ignorują najnowsze standardy sieciowe, aby „poprawnie” wyświetlać strony, które zostały utworzone zgodnie z zasadami obowiązującymi w latach 90.
Tryb standardowy
W trybie standardowym przeglądarki w miarę swoich możliwości próbują przetwarzać dobrze zbudowane dokumenty zgodnie ze standardami. W HTML5 tryb ten nazywa się „trybem bez dziwactw”.
Tryb prawie standardowy
Firefox, Safari, Chrome, Opera (od 7.5) i IE 8 mają dodatkowo tryb zwany „trybem prawie standardowym”, w którym algorytmy określania wysokości komórek tabeli są zaimplementowane w tradycyjny sposób, niezgodny ze specyfikacją CSS 2. W HTML5 tryb ten nazywa się „trybem umiarkowanych dziwactw”.
